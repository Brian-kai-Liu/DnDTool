# DnD 开发优先级路线图

目标：先把“生物部署到网格地图”这条主线做稳，再逐步扩展到完整遭遇与战斗流程。

## 当前总原则

1. 先数据，后交互。
2. 先最小闭环，后完整规则。
3. 预览态和真实数据态分离。
4. 生物模板和战场实例分离。

## 优先级

### P0 基础数据结构

1. 明确生物模板与战场实例的区分。
2. 明确格子坐标、占格、选中、地形、事件的边界。
3. 增加遭遇/部署层数据结构，避免直接把生物塞进格子标记里。
4. 统一运行时模型与保存 DTO 的映射方式。
5. 明确战场实例必须保存“生物模板来源 + 部署格子位置”。
6. 明确战场实例需要支持 DM 局部修改，并保存修改结果。
7. 明确战场实例支持“未上场 / 条件触发后上场”的生命周期。

### P1 生物部署 MVP

1. 左侧生物卡拖拽到地图。
2. 松手后落到目标格子。
3. 形成可视化 token。
4. 支持基础占格与碰撞提示。

### P2 地图上操作

1. 已部署生物继续拖动。
2. 支持移除与重新部署。
3. 支持多格体型与占位高亮。

### P3 遭遇运行

1. 先攻顺序。
2. 当前 HP / 状态 / 死亡标记。
3. 战斗备注与 DM 快速操作。

### P4 持久化与恢复

1. 保存部署位置、单位状态、遭遇信息。
2. 重开章节后完整恢复。
3. 兼容旧数据。

## 建议执行顺序

1. 先做 P0。
2. 再做 P1。
3. 然后补 P2。
4. 最后推进 P3 和 P4。

## 当前下一步

先落地 P0：数据结构和职责边界先定死，再开始拖拽部署。

## P0 已确认原则

1. 战场实例不写入 GridCells。
2. 战场实例位置按格子坐标保存，不按 UI 像素坐标保存。
3. 战场实例至少要持久化保存：
   - 实例 ID
   - 来源生物 ID
   - 锚点格子坐标
4. 战场实例需要保存 DM 对实例的调整结果，不影响原始生物模板。
5. 生物不一定开场就在地图中，需要支持“延迟登场”。
6. 场景中的角色需要区分“常驻实例”和“事件驱动实例”。
7. 战场实例需要有可被 DM 与事件共同控制的“是否激活出现”状态。
